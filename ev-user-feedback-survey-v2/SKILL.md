# ev-user-feedback-survey-v2：用户反馈调研四阶段流水线

## 核心设计理念

```
用户发起调研任务
       ↓
  ┌─────────────────┐
  │  Stage0 范围确认  │  ⏸️ 确认时间范围/功能域/竞品
  └────────┬────────┘
           ↓ Gate0 确认
  ┌─────────────────┐
  │  Stage1 收集员   │  多平台数据抓取
  └────────┬────────┘
           ↓ Gate1
  ┌─────────────────┐
  │  用户补充窗口     │  ⏸️ 等待用户（链接/文本/无补充则继续）
  └────────┬────────┘
           ↓ Gate2
  ┌─────────────────┐
  │  Stage2 分析师   │  AI独立分析 或 用户+AI协作分析
  └────────┬────────┘
           ↓ Gate3
  ┌─────────────────┐
  │  Stage3 报告员  │  飞书文档写入 + 完成汇报
  └────────┬────────┘
             ↓
         任务完成
```

## 触发词

"做用户调研"、"用户反馈调研"、"口碑调研"、"车主评价调研"、"产品体验调研"、"帮我分析用户反馈"、"启动调研流程"

## Stage0：调研范围确认（强制前置步骤）

**职责：** 在正式启动信息收集之前，与用户确认调研的时间边界、功能边界和竞品范围。这是调研质量的源头把控，不得跳过。

**执行文件：** `references/00-scope-confirm.md`

### 确认四要素

| 要素 | 说明 | 示例 |
|------|------|------|
| **时间范围** | 数据采集的时间边界 | 2024年1月 ～ 2025年12月 |
| **功能域** | 本次调研聚焦的功能/系统/服务 | 空调系统、售后服务、语音交互 |
| **品牌/产品** | 主调研对象（单一或多个） | 领克全系、问界M7 |
| **竞品范围** | 是否需要覆盖竞品对比数据 | 理想L系列、蔚来ES系列 |

### 默认规则

若用户未明确指定，则采用以下默认：

| 要素 | 默认值 |
|------|--------|
| 时间范围 | 最近24个月 |
| 功能域 | 用户指定品牌/产品的整体用户体验 |
| 品牌/产品 | 用户明确提出的单一品牌/产品 |
| 竞品范围 | 无（单品牌调研），除非用户主动提出 |

### 脱敏处理

若用户提供的信息涉及内部产品代号、项目名称，应在调研启动前完成脱敏替换，并在报告中使用公开名称。

**Gate0 征询单：**

```
━━━━━━━━━━━━━━━━━━━━━━
【Gate0 / 调研范围确认】

在正式启动之前，请确认以下调研范围：

1️⃣ 时间范围
   我理解的时间范围：[最近24个月 / 用户指定]
   如有调整请告知起止时间

2️⃣ 功能域 / 功能范围
   我理解的本次调研聚焦于：[整体用户体验 / 用户指定]
   如需聚焦特定功能请补充（如：空调系统 / 售后服务 / 语音交互）

3️⃣ 品牌 / 产品
   主调研对象：[用户指定品牌+产品]

4️⃣ 竞品范围
   是否需要覆盖竞品：[否 / 是，请指定]

5️⃣ 数据来源偏好
   是否有特别关注平台：[无 / 是，请指定]

→ 回复「确认」：按以上范围正式启动
→ 回复「调整+具体内容」：我会按您的要求修改后启动
━━━━━━━━━━━━━━━━━━━━━━
```

## Stage1：信息收集

**职责：** 调用全平台搜集能力，按 Stage0 确认的范围，抓取公开用户反馈数据。

**执行文件：** `references/01-collector.md`

**搜集平台能力矩阵：**

| 平台 | 帖子正文 | 用户评论 | 原始链接 | 备注 |
|------|---------|---------|---------|------|
| 汽车之家 | ✅ | ✅ | ✅ | 首选平台之一 |
| 懂车帝 | ✅ | ✅ | ✅ | 首选平台之一 |
| 太平洋汽车 | ✅ | ✅ | ✅ | 首选平台之一 |
| 微博 | ✅ | 部分 | ✅ | 部分内容可抓取 |
| 知乎 | ✅ | 部分 | ✅ | 含问答结构 |
| 今日头条 | ✅ | ✅ | ✅ | 部分需登录 |
| 微信文章 | ✅ | ❌ | ✅ | via URL直读 |
| 小红书 | ⚠️ 需用户手动 | ❌ | ✅ via Google | 需方案二 |

**输出：** 原始数据文件，含完整可溯源链接，严格按 Stage0 确认的范围采集。

**Gate1 征询单：**

```
━━━━━━━━━━━━━━━━━━━━━━
【Gate1 / 信息收集完成】

✅ 已完成（按Gate0确认范围）：
- 调研范围：时间[XXX] / 功能域[XXX] / 品牌[XXX]
- 搜集平台：N个
- 有效数据：N条
- 覆盖品牌/产品：XXX

⚠️ 请确认：
1. 以上数据覆盖是否符合您的调研需求？
2. 是否有需要补充的平台或品牌？（请提供链接或关键词）
3. 是否需要我补充其他竞品数据？

→ 回复「继续」：进入分析阶段（无需补充）
→ 回复「补充+内容」：我将处理您提供的链接/文本后再继续
━━━━━━━━━━━━━━━━━━━━━━
```

## Stage2：协作分析

**职责：** 对 Stage1 数据 + 用户补充输入做情感分析、观点提炼、洞察挖掘。

**执行文件：** `references/02-analyst.md`

**双路径决策：**

| 情况 | 处理方式 |
|------|---------|
| Gate1 用户无补充 | AI 独立分析 Stage1 全量数据 |
| Gate1 用户有补充 | AI 先处理用户输入（URL读取/文本解析），再与 Stage1 数据合并分析 |

**输出：** 结构化分析结果（洞察清单 + 情感分类 + 关键引用）

**Gate2 征询单：**

```
━━━━━━━━━━━━━━━━━━━━━━
【Gate2 / 分析完成】

✅ 已完成：
- 正面反馈：N条（N%）
- 质疑/负面反馈：N条（N%）
- 中性反馈：N条（N%）
- 核心洞察：N个

⚠️ 请确认：
1. 以上洞察是否符合您的预期方向？
2. 是否有需要特别关注的点？（请补充）
3. 报告结构有无特别要求？

→ 回复「继续」：进入报告产出
→ 回复「补充+内容」：我会处理后再进入报告
━━━━━━━━━━━━━━━━━━━━━━
```

## Stage3：报告产出

**职责：** 将分析结果整理为结构化报告，写入飞书文档，完成交付。

**执行文件：** `references/03-reporter.md`

**报告结构（标准模板）：**

```
1. 数据来源总览（含调研范围说明）
2. 产品/品牌背景
3. 用户反馈分析（正面/质疑/负面分类）
4. 竞品对比（如适用）
5. 关键洞察（≤5个）
6. 数据局限性
7. 后续建议
```

## 质量保障原则

- **零编造**：数据必须有明确来源，无来源数据必须标注"推测"
- **零跳过**：Gate 是信任机制，不得跳过
- **零删减**：报告模块不得无故省略
- **可溯源**：所有数据点必须有原始链接支撑
- **范围先行**：Stage0 范围确认是一切的前提，不得跳过
