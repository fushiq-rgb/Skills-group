# 013-prd-agent：PRD需求工作流

## 整体架构

```
用户输入原始素材
       ↓
┌─────────────────────┐
│  Agent1：会议转录整理   │ ← 仅读取原始素材
│  写入：Agent1输出分区   │
└─────────────────────┘
       ↓
┌─────────────────────┐
│  Agent2：业务意图翻译   │ ← 仅读取Agent1分区
│  写入：Agent2输出分区   │
└─────────────────────┘
       ↓
┌─────────────────────┐
│  Agent3：PRD结构化写作  │ ← 仅读取Agent2分区
│  写入：Agent3输出分区   │
└─────────────────────┘
       ↓
┌─────────────────────┐
│  Agent4：评审挑战模拟    │ ← 仅读取Agent3分区
│  写入：Agent4输出分区   │
└─────────────────────┘
       ↓
    飞书文档（完整交付物）
```

## 飞书文档协作规则（全局统一）

1. **单文档传递**：全流程共用1份飞书文档，通过「分区隔离」传递信息，不额外创建文件
2. **分区固定**：

| 分区 | 内容 | 可读写Agent |
|------|------|------------|
| 原始素材区 | 用户提供的原始输入材料 | 用户（写入）/ Agent1（读取）|
| Agent1输出区 | 事实整理员输出 | Agent1（写入）/ Agent2（读取）|
| Agent2输出区 | 意图翻译官输出 | Agent2（写入）/ Agent3（读取）|
| Agent3输出区 | PRD初稿 | Agent3（写入）/ Agent4（读取）|
| Agent4输出区 | 评审材料 | Agent4（写入）/ 用户（读取）|

3. **禁止跨区操作**：每个Agent只能读取上游分区、写入自身分区，不得修改其他分区
4. **脱敏处理**：所有敏感信息在写入文档前必须完成脱敏，禁止原始敏感数据进入流程

## 执行触发

当用户说"开始PRD流程"或类似表达时，执行以下步骤：

### Step 1：确认飞书文档
- 询问用户提供飞书文档链接（已有文档则继续，无文档则按skill-creator规范创建）
- 在文档中按上表建立5个分区，每个分区以标题明确标注

### Step 2：读取各角色提示词
依次读取以下reference文件，获取对应Agent的详细执行指令：
```
references/01-recorder.md    → Agent1执行指令
references/02-intent-translator.md  → Agent2执行指令
references/03-prd-writer.md → Agent3执行指令
references/04-reviewer.md    → Agent4执行指令
```

### Step 3：顺序执行4个Agent（单会话内顺序调用）

**Agent1执行**：
- 按 01-recorder.md 指令处理用户原始素材
- 完成后输出执行完成确认
- 确认全部6个强制输出项均已写入飞书文档Agent1分区

**Agent2执行**：
- 按 02-intent-translator.md 指令读取Agent1输出
- 完成后输出执行完成确认
- 确认全部5个强制输出项均已写入飞书文档Agent2分区

**Agent3执行**：
- 按 03-prd-writer.md 指令读取Agent2输出
- 输出完整PRD初稿（六大核心模块）
- 完成后输出执行完成确认
- 确认全部7个强制输出项均已写入飞书文档Agent3分区

**Agent4执行**：
- 按 04-reviewer.md 指令读取Agent3 PRD
- 完成多角色评审挑战
- 完成后输出执行完成确认
- 确认全部5个强制输出项均已写入飞书文档Agent4分区

### Step 4：全流程完成汇报
向用户汇报：
- 飞书文档链接
- 4个Agent的执行完成状态（全部✅）
- 各分区核心产出摘要

## 质量保障原则

- **零推测**：每个输出项必须有明确来源依据，不得自行填补空白答案
- **零遗漏**：6大模块/输出项必须完整，禁止标注"待定"
- **零跨区**：各Agent严格遵守分区边界，不读取非必要分区内容
- **脱敏前置**：敏感信息在写入文档前必须完成替换，不存储原始数据
- **完成确认规范化**：每个Agent输出末尾必须包含标准化完成声明，不得删减
